# Security

## Security Philosophy

Безопасность закладывается на этапе проектирования.

Мы не добавляем безопасность позже.

Каждая новая функция должна проходить проверку на:

* безопасность
* приватность
* соответствие законодательству РФ

---

# Threat Model

Основные угрозы:

1. Боты
2. Спамеры
3. Парсеры базы кандидатов
4. Взлом аккаунтов
5. Массовая регистрация
6. DDoS
7. Утечка персональных данных
8. Уязвимости API
9. Инсайдерские угрозы
10. Ошибки разработчика

---

# Authentication

Используется:

Supabase Auth

---

Разрешённые способы входа:

* Email + Password
* Magic Link
* Telegram Login (будущее)

---

Запрещено:

* хранение паролей в собственной БД
* собственная реализация авторизации

---

12+ символов

---

Обязательно:

* буквы
* цифры

# Rate Limiting

Обязательно.

---

Регистрация:

5 попыток в час

на IP

---

Логин:

10 попыток в час

на IP

---

Отправка сообщений:

не более:

30 сообщений в минуту

---

Создание вакансий:

не более:

20 вакансий в сутки

---

# Bot Protection

Использовать:

Cloudflare Turnstile

---

Причины:

* бесплатно
* лучше UX чем reCAPTCHA
* защита от ботов

---

Проверять капчу:

* регистрация
* логин
* отправка жалоб
* массовые действия

---

# Anti Spam

Запрещать:

* массовые сообщения
* ссылки в первых сообщениях
* повторяющиеся сообщения

---

Автоматическая блокировка:

при подозрении на спам

---

# Anti Scraping

Защита от массового сбора данных.

---

Ограничить:

* количество просмотров

* скорость запросов

Логировать:

* подозрительные IP
* аномальную активность

# API Security

Все API защищаются.

---

Требования:

JWT Authentication

RLS Policies

Input Validation

Output Filtering

Rate Limiting

---

---

* права доступа
* роль пользователя
* принадлежность данных

# RLS Policies

Версия: 1.0

Цель:

Настроить Row Level Security таким образом, чтобы пользователь имел доступ только к своим данным.

Все ограничения должны работать на уровне PostgreSQL независимо от фронтенда.

---

# Общие правила

RLS должен быть включен для ВСЕХ таблиц.

```sql
ALTER TABLE table_name ENABLE ROW LEVEL SECURITY;
```

---

Запрещено:

* использовать service_role во frontend
* отключать RLS ради удобства
* выдавать SELECT на все записи

# Дополнительные ограничения

## Ограничение размера сообщений

messages

Максимум:

```text
2000 символов
```

---

## Ограничение количества жалоб

Не более:

```text
20 жалоб в сутки
```

на пользователя.

---

## Ограничение свайпов

Free:

```text
50 свайпов в сутки
```

---

Pro:

```text
без ограничений
```

---

# Защита от массового парсинга

Добавить rate limit:

```text
100 запросов / минута
```

на пользователя.

---

# Защита от перебора UUID

Все запросы должны проходить через RLS.

Даже если пользователь знает UUID записи:

доступ запрещен.

---

# Обязательные проверки

Перед релизом проверить:

* пользователь не видит чужие сообщения
* пользователь не видит чужие жалобы
* пользователь не может редактировать чужой профиль
* работодатель не может менять чужие вакансии
* кандидат не может создавать вакансии
* нельзя получить данные напрямую через REST API Supabase
* нельзя читать таблицы без авторизации

---

# Security Rule

Если возникает выбор между:

* удобством
* безопасностью

всегда выбирать безопасность.
